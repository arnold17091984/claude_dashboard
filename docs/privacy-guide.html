1. このダッシュボードは何をするもの？

チーム全体のClaude Code 利用コストと利用傾向を把握するためのツールです。

「誰が何を書いたか」ではなく、「チーム全体でどのくらい使っているか」を見るためのものです。

2. 送信されるデータ（これだけ）

データ例目的
セッションIDabc-123-defセッション重複排除
プロジェクト名my-appプロジェクト別集計
プロジェクトパス~/repos/my-appプロジェクト特定
※ユーザー名はマスク済み
Gitブランチfeature/login作業内容の大まかな把握
使用モデルclaude-sonnet-4-6コスト分析
ツール名Read, Edit, Bashツール利用傾向（名前だけ）
トークン数入力: 50,000コスト計算
利用時間45分利用傾向

3. 絶対に送信されないデータ

項目送信？
会話の内容（あなたの質問・Claudeの回答）❌ 送信されない
コードの中身❌ 送信されない
ファイルの内容❌ 送信されない
Bashコマンドの実行結果❌ 送信されない
ブラウザ履歴❌ 送信されない
メール・写真・個人ファイル❌ 送信されない
PCのユーザー名❌ 送信されない（マスク処理済み）
💡 このダッシュボードは「経費精算書」のようなものです。
「何にいくら使ったか」は見えますが、「何を買ったかの中身」は見えません。

4. データの流れ

5. 管理者に見えるもの vs 見えないもの

✅ 管理者が見えるもの
  • チーム全体のコスト推移
  • ユーザーごとの利用時間
  • よく使われるツール・モデル
  • プロジェクト別の利用状況
❌ 管理者に見えないもの
  • あなたが何を質問したか
  • あなたが何のコードを書いたか
  • あなたのPC上のファイル
  • Claude Code以外のPC利用状況

6. あなたのコントロール

やりたいこと方法
送信を止めたいsettings.json からフックを削除するだけ
何が送信されるか確認したいフックは curl コマンドなので -v をつけて中身を確認可能
ニックネームを変えたいダッシュボードの設定画面から変更
APIキーを無効にしたいダッシュボードで再発行すれば旧キーは無効に
データの削除を依頼したい管理者に依頼すればアカウントとデータを削除可能
🔑 ポイント

フックの設定はあなた自身が行います。管理者がリモートで勝手にインストールすることはできません。
また、いつでも自分の判断で送信を停止できます。

7. セットアップ手順

ステップ 1：ダッシュボードにアカウント作成

  1. ダッシュボードの URL にアクセス
  2. 「アカウント作成」をクリック
  3. 管理者から受け取った招待コードを入力
  4. お好みのニックネームとパスワードを設定
  5. 個人APIキー（dk_xxxxxxxxxx）が自動発行されます

ステップ 2：Claude Code のフック設定

~/.claude/settings.json に以下を追加してください：

{
  "hooks": {
    "SessionEnd": [
      {
        "command": "curl -s -X POST https://dashboard.example.com/api/v1/ingest/session \
          -H 'Authorization: Bearer dk_あなたのAPIキー' \
          -H 'Content-Type: application/json' \
          -d '{統計データ}'"
      }
    ]
  }
}
📝 補足

具体的なフック設定テンプレートは管理者から配布されます。
上記はイメージ図であり、実際のコマンドはテンプレートをコピー＆ペーストするだけです。

ステップ 3：完了

以上で設定は完了です。あとはClaude Codeのセッション終了時に、自動的に統計データが送信されます。

8. よくある質問

質問回答
強制ですか？いいえ。フック設定は任意です。
個人PCでも安全ですか？はい。Claude Codeのセッション統計のみ送信されます。
PC内のファイルや個人データには一切アクセスしません。
サボっているとバレますか？利用時間は見えますが、目的はコスト管理であり
勤務評価には使用しません。
退職時にデータは削除されますか？管理者に依頼すれば削除されます。
送信データを事前に確認できますか？はい。フックはcurlコマンドなので、
送信内容は完全に透明です。

9. 技術的な安全対策

  • 通信の暗号化：HTTPS（TLS）で暗号化されて送信されます
  • 個人APIキー認証：APIキーがないとデータ送信できません
  • パスのマスキング：/Users/tanaka/repos/app → ~/repos/app に自動変換
  • ニックネーム表示：ダッシュボード上ではPCユーザー名ではなくニックネームで表示
  • レート制限：不正な大量リクエストを防止
  • セキュリティヘッダー：CSP, HSTS 等の標準的なWebセキュリティ対策を実装
